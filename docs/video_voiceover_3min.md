# Three-Minute Build Week Voiceover

Approximate runtime: 2:55 to 3:05 at a calm speaking pace.

## Script

Most analytics tools begin with an output.

A dashboard.

A model summary.

A report.

Analytics Workstation starts one step earlier.

It starts with uncertainty.

This project asks a simple question:

What if AI did not just generate an answer?

What if it helped conduct an investigation?

The demo uses a deterministic synthetic enrollment dataset.

The ground truth exists for testing, but the demo does not use it.

The workstation has to reason from the evidence it can inspect.

Before anything runs, the system performs preflight.

It checks the dataset, analytical services, report browser, replay path, and runtime contracts.

If something is missing, the app should say so.

Now the investigation begins.

The first belief is intentionally incomplete.

One marketing channel appears inefficient.

In a normal dashboard, that might become a recommendation too quickly.

Here, it becomes a hypothesis.

The workstation keeps competing explanations visible.

Search may be saturated. Creative may be aging. Operations may be slowing conversion.

The system chooses evidence before conclusion.

Deterministic services do the analytical work.

EDA checks the data. Regression gives a baseline driver story. SHAP evidence sits behind an approval step.

The AI is not inventing the evidence.

It helps frame, synthesize, and explain what the evidence means.

As evidence accumulates, the belief changes.

That is the important part.

The system changes its mind because the evidence changed.

The recommendation becomes more specific.

Do not simply cut demand generation.

Improve operational throughput.

Refresh aging creative.

Tune saturated Search spend.

Preserve the remaining uncertainty.

At this point, the important question is not what the system recommended.

The important question is:

Why should I believe this?

Analytics Workstation answers that question directly.

It shows the initial belief, the evidence discovered, how the belief changed, diagnostics, methodology, limitations, and remaining uncertainty.

Then it performs one final check.

The workstation challenges its own conclusion.

It asks what could still be wrong.

It looks for contradictory evidence, preserves alternative explanations, states assumptions, and assesses decision readiness.

This is the product's central idea.

AI should not replace analytical judgment with a fluent answer.

It should help preserve the reasoning path that makes judgment possible.

Analytics Workstation is both software and a development experiment.

AI-assisted implementation made it possible to build quickly.

The project used that speed to ask harder questions about evidence, governance, trust, and product experience.

The result is not a general autonomous analyst.

It is a bounded demonstration of a different pattern:

investigate first,

preserve the evidence,

verify the claim,

and challenge the recommendation before asking anyone to trust it.
